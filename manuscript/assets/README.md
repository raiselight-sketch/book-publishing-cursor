# 책 삽화·사진·AI 이미지 워크플로

이 책은 **사진·도판·AI 생성 이미지**를 넣는다. 여기는 **원본 파일을 모아 두는 곳**이며, 한글/인디자인으로 옮길 때 같은 경로·이름을 유지하면 편하다.

## 폴더 역할

| 경로 | 용도 |
|------|------|
| `photos/` | 카메라·스캔 **실사**. 원본 해상도 유지. |
| `ai-generated/` | DALL·E, Midjourney, Imagen, SD 등 **AI 출력**. |
| `figures/` | 다이어그램·표 스냅·아이콘 등 **편집 확정본**(PNG/WebP 등). |

선택: `exports-for-print/` — 출판사 제출용 CMYK 변환 후(디자이너 작업물)

## 파일 이름 규칙 (권장)

```
ch{챕터번호}-{역할}-{순번}.{확장자}
예: ch03-photo-01.jpg, ch05-ai-02.png, ch07-figure-01.png
```

챕터가 아직 없으면 `book-photo-01`처럼 임시 번호를 쓰고, 목차 확정 후 이름만 바꿔도 된다.

## 본문(마크다운)에서 넣는 법

같은 저장소 기준 **상대 경로**를 쓴다.

```markdown
![3장 — 놀이 장면 설명](assets/photos/ch03-photo-01.jpg)
```

캡션 문구는 본문에 짧게 두고, **그림 목록**은 `outline.md` 또는 별도 `캡션-목록.md`에 표로 정리한다.

## 인쇄용 해상도 (가이드)

- **단행본 컬러 삽화**: 긴 변 기준 **최소 2400px 내외** 이상이 안전한 경우가 많다(판형·출판사 지침이 우선).  
- **흑백 본문 사진**: 300dpi 기준으로 **인쇄 크기의 1.2~1.5배** 여유.  
- **전자책 위주**: 가로 1600~2400px 정도로도 무방한 경우가 많다.

최종 수치는 **인쇄소·출판사 스펙**을 받은 뒤 맞춘다.

## AI 이미지를 쓸 때 꼭 할 일

1. **이용약관**: 상업적 도서 출판에 **허용되는지** 툴마다 확인한다.  
2. **근거 남기기**: 아래 매니페스트에 **프롬프트 요약·도구명·날짜·버전**을 적어 둔다(출판사·윤리 문의 대비).  
3. **표기**: 책 끝 또는 그림 아래에 “일부 삽화 AI 보조 생성” 등 **출판사가 요구하는 표기**를 맞춘다.  
4. **사람·브랜드**: 실존 인물·특정 로고를 닮게 만들지 않도록 프롬프트를 조정한다.
