# 책 집필 & 출판 완전 정복 스킬

## 이 스킬의 목적

단순히 글을 잘 쓰는 것이 아니라, **독자에게 읽히고, 시장에서 팔리고, 세상에 영향을 미치는 책 한 권**을 완성하는 것. 기획 → 집필 → 퇴고 → 출판 → 유통의 전 과정을 지원한다.

---

## PHASE 0: 현재 단계 파악

사용자와 처음 대화할 때 반드시 먼저 파악하라:

1. **단계**: 아이디어만 있음 / 목차 있음 / 초고 있음 / 퇴고 중 / 출판사 찾는 중
2. **장르**: 신학·설교집 / 신앙 에세이 / 간증 / 자기계발 / 실용서 / 소설 / 논문형
3. **독자 대상**: 교인 / 청년 / 목회자 / 일반 대중 / 특정 커뮤니티
4. **분량 목표**: 단행본(200~350p) / 소책자(100p 이하) / 시리즈
5. **출판 방식**: 전통 출판사 / 자비 출판 / POD / 전자책
6. **삽화·이미지**: 이 책에 **사진·도판·AI 생성 그림**이 들어가는지, 인쇄(컬러 면수) vs 전자책 위주인지, 실사와 AI의 **비율·톤**(밝음/차분 등)을 확인하라.

## PHASE 1: 책 기획 (Book Concept)

### 1-1. 핵심 한 문장 (Core Sentence) 도출

책의 존재 이유를 한 문장으로 정의하라:
> "이 책은 [독자]가 [문제/상황]에서 [변화/결과]를 얻도록 돕는다."

예시:
- "이 책은 **지친 그리스도인**이 **신앙의 피로감**에서 **생명의 기쁨**을 회복하도록 돕는다."
- "이 책은 **자녀를 키우는 부모**가 **가정 예배**를 **자연스럽게 세우도록** 안내한다."

### 1-2. 책의 세 기둥 (Three Pillars)

모든 좋은 책은 세 가지를 균형 있게 담는다:

| 기둥 | 질문 | 역할 |
|------|------|------|
| **문제(Problem)** | 독자가 왜 이 책을 집어 드는가? | 공감 / 필요 유발 |
| **통찰(Insight)** | 저자만이 줄 수 있는 관점은? | 차별화 / 신뢰 구축 |
| **변화(Transformation)** | 읽고 나면 무엇이 달라지는가? | 가치 / 구매 동기 |

### 1-3. 책 제목 만들기

좋은 제목의 조건:
- **명확성**: 무슨 책인지 3초 안에 알 수 있어야 한다
- **매력**: 손이 가야 한다 (감성 / 호기심 / 약속)
- **검색 가능성**: 온라인 검색에서 발견될 수 있어야 한다

제목 구조 패턴:
```
[명사형] → "하나님의 시간표"
[동사형] → "지금 당장 무릎 꿇어라"
[질문형] → "당신은 왜 교회를 떠났는가?"
[대비형] → "고통이 축복이 될 때"
[숫자형] → "목사님이 말하지 않는 신앙의 7가지 비밀"
[부제 활용] → "메인 제목: 감성적 / 부제: 설명적"
```

## PHASE 2: 구조 설계 (Architecture)

### 2-1. 목차 설계 원칙

**황금 목차의 구조:**

```
프롤로그 (저자의 이야기 / 이 책을 쓴 이유)
  ↓
1부: 문제를 직시하다 (공감 / 현실 진단)
  ↓
2부: 진리를 만나다 (통찰 / 성경적·신학적 기반)
  ↓
3부: 변화를 살아내다 (적용 / 실천 / 간증)
  ↓
에필로그 (독자에게 보내는 편지 / 소망)
부록 / 참고문헌 (선택)
```

**챕터 하나의 구조 (권장):**
```
1. 훅(Hook): 이야기나 질문으로 시작
2. 문제 제시: 이 챕터가 다루는 핵심 긴장
3. 성경/사례/논거: 신뢰할 수 있는 근거
4. 통찰 전달: 저자의 핵심 메시지
5. 적용: 독자가 할 수 있는 구체적 행동
6. 마무리: 다음 챕터로 자연스럽게 연결
```

### 2-2. 챕터 수와 분량 가이드

| 책 유형 | 권장 챕터 수 | 챕터당 분량 | 총 원고량 |
|---------|------------|-----------|---------|
| 신앙 에세이 | 12~20개 | 2,500~4,000자 | 6~8만 자 |
| 설교집 | 20~30편 | 3,000~5,000자 | 8~12만 자 |
| 실용서/자기계발 | 10~15개 | 4,000~6,000자 | 8~10만 자 |
| 신학 전문서 | 8~12개 | 6,000~10,000자 | 10~15만 자 |
| 간증/자서전 | 10~18개 | 3,000~5,000자 | 6~9만 자 |

### 2-3. 삽화·사진·AI 이미지 (도판 포함)

이 저장소에서 **집필 중인 책에 시각 자료(사진·도판·AI 이미지)가 포함된다**고 가정하고 다음을 지킨다.

- **원본 위치**: `manuscript/assets/` — 세부 규칙·해상도·저작권 체크리스트는 `manuscript/assets/README.md`를 따른다.
- **집필 시**: 마크다운에서는 `![캡션](assets/photos/ch01-photo-01.jpg)`처럼 **상대 경로**로 삽입 위치를 표시한다.
- **AI 이미지**: 생성 툴의 **상업 이용 허용**, 책 속 **표기 문구**를 출판 전에 확정하도록 안내한다. 파일마다 `IMAGE-MANIFEST`에 도구명·프롬프트 요약·날짜를 남기도록 권한다.
- **캡션·그림 목록**: 챕터별로 “그림 3-1 …” 형태를 통일하고, 퇴고 단계에서 번호·캡션 오류를 점검한다.

## PHASE 3: 집필 (Writing)

### 3-1. 집필 환경 세팅

**일일 집필 루틴 (권장):**
```
① 기도 / 묵상 (10분) — 영적 정렬
② 어제 쓴 마지막 단락 다시 읽기 (5분) — 흐름 연결
③ 목표 분량 집필 (60~90분) — 방해 없는 집중
④ 메모·아이디어 캡처 (5분) — 떠오른 것들 기록
```

**최소 일일 목표**: 1,000~1,500자 (한 달이면 챕터 하나)

### 3-2. 초고 집필 원칙

> "초고는 완벽하지 않아도 된다. 쓰여져야 한다."

- **비판 금지 원칙**: 초고를 쓸 때는 편집자 모드를 끈다
- **속도 우선**: 멈추지 말고 끝까지 쓴다
- **괄호 활용**: 나중에 채울 내용은 [나중에 추가] 표시
- **완성이 완벽을 이긴다**: 300쪽의 평범한 원고가 50쪽의 완벽한 원고보다 낫다

### 3-3. 글쓰기 문체 원칙

**읽히는 글의 5원칙:**

1. **짧은 문장**: 한 문장에 하나의 생각. 60자 이내 권장
2. **구체적 이야기**: 추상 개념은 반드시 이야기나 사례로 보여준다
3. **독자에게 말하기**: "우리는" 보다 "당신은"이 더 강하다
4. **리듬과 변화**: 긴 문장 다음엔 짧은 문장. 단락 길이를 다양하게
5. **여백의 미**: 단락은 5~7문장을 넘기지 않는다

**피해야 할 것:**
- 설교 투 그대로 옮기기 (책은 설교가 아니다)
- 지나친 신학 용어 (독자가 전문가가 아닐 수 있다)
- "~하는 것이다" 반복 (단조로움)
- 인용만 많고 저자 목소리 없음

### 3-4. 챕터별 집필 체크리스트

```
□ 훅이 있는가? (첫 문단에서 독자를 잡는가)
□ 이 챕터의 핵심 메시지가 하나인가?
□ 성경 본문 / 사례 / 근거가 있는가?
□ 저자의 개인 경험이 녹아 있는가?
□ 독자를 위한 적용이 있는가?
□ 다음 챕터가 궁금해지는 마무리인가?
```

## PHASE 4: 퇴고 (Revision & Editing)

### 4-1. 3단계 퇴고 시스템

**1차 퇴고 — 구조 퇴고 (매크로)**
- 전체 흐름이 논리적인가?
- 불필요한 챕터는 없는가?
- 챕터 순서를 바꿔야 하는가?
- 반복되는 내용은 없는가?

**2차 퇴고 — 문장 퇴고 (마이크로)**
- 어색한 문장 다듬기
- 불필요한 단어 제거 ("사실", "정말", "매우", "아주" 등)
- 수동태 → 능동태 변환
- 전문 용어 → 쉬운 표현

**3차 퇴고 — 교정 (Proofreading)**
- 맞춤법 / 띄어쓰기
- 성경 구절 정확성 확인
- 인명 / 지명 표기 통일
- 참고문헌 정확성

### 4-2. 퇴고 도구

```bash
# 맞춤법 검사 활용
- 부산대 한국어 맞춤법 검사기 (http://speller.cs.pusan.ac.kr)
- 네이버 맞춤법 검사기
- Microsoft Word 맞춤법 기능
```

### 4-3. 베타 리더 (Beta Reader) 활용

출판 전 3~5명의 베타 리더에게 초고 검토 요청:
- 교인/신도 1명 (일반 독자 관점)
- 동료 목회자 1명 (신학적 검토)
- 비신자 1명 (접근성 확인)
- 편집자 경험자 1명 (구조/문장)

**베타 리더에게 물을 것:**
1. 읽다가 지루해진 부분이 있었나?
2. 이해가 안 된 표현이나 단락이 있나?
3. 가장 인상 깊었던 챕터는?
4. 누구에게 이 책을 추천하겠는가?

## PHASE 5: 출판 준비 (Publishing Prep)

### 5-1. 출판 방식 선택

| 방식 | 장점 | 단점 | 적합 대상 |
|------|------|------|---------|
| **전통 출판사** | 신뢰도·유통력 높음, 비용 없음 | 수익 적음(10%), 결정권 제한 | 대중 독자 겨냥, 첫 책 |
| **자비 출판** | 완전한 통제권, 빠른 출판 | 비용 부담(300~800만원), 유통 취약 | 교회 내 배포, 선물용 |
| **POD(주문형인쇄)** | 재고 없음, 소량 가능 | 단가 높음, 인지도 낮음 | 소규모 독자, 실험적 |
| **전자책(e-book)** | 비용 최소, 즉시 출판 | 수익 낮음, 종이책 선호 시장 | 부가 수익, 해외 독자 |

### 5-2. 출판사 투고 전략

**투고 패키지 구성:**

```
1. 기획안 (2~3페이지)
   - 책 소개 (한 단락)
   - 핵심 메시지
   - 독자 대상
   - 경쟁 도서와의 차별점
   - 목차 (완성된 것)
   - 저자 소개 (약력 + 플랫폼)

2. 샘플 챕터 (2~3개)
   - 가장 잘 쓴 챕터 선택
   - 완성도 높은 상태여야 함

3. 저자 플랫폼 자료
   - SNS 팔로워 수
   - 유튜브/팟캐스트 구독자
   - 강연·설교 청중 규모
   - 기존 출판물
```

**투고 적합 출판사 (기독교 분야):**
- 규장, 두란노, 생명의말씀사, IVP, 복있는사람, 넥서스CROSS, 홍성사, 대한기독교서회

**일반 자기계발/에세이:**
- 위즈덤하우스, 북하우스, 김영사, 21세기북스, 다산북스

### 5-3. 원고 제출 형식

```
파일 형식: HWP 또는 DOCX
폰트: 바탕체 또는 맑은 고딕 10pt
줄간격: 160~180%
여백: 기본값 (상하좌우 25mm)
페이지 번호: 필수
파일명: [책제목_저자명_원고].hwp
```

## PHASE 6: 출판 후 마케팅

### 6-1. 출판 전 버즈(Buzz) 만들기

- 집필 과정 SNS 공유 ("책 쓰는 목사" 브랜딩)
- 챕터 일부를 블로그/유튜브로 먼저 공개
- 북카페 / 교회 서평단 모집
- 출판 기념 강연 기획

### 6-2. 출판 직후 90일 집중 마케팅

```
1주: 출판 알림 (SNS, 교회 광고, 메일링)
2주: 서평 이벤트 (독자 후기 공모)
3~4주: 언론 보도 / 인터뷰 (크리스천 매체)
2개월: 강연·북토크 진행
3개월: 재쇄 여부 출판사와 협의
```

### 6-3. 저자 플랫폼 구축

책은 한 권이 아니라 **저자 브랜드**의 시작이다:

```
블로그/브런치 → 책 내용 연재, SEO 활용
유튜브/팟캐스트 → 책 내용 강의화
인스타그램 → 책 문구 카드뉴스
교회/강단 → 설교와 책 연계
뉴스레터 → 독자와 직접 소통
```

## 에이전트가 할 수 있는 것

이 스킬 안에서 에이전트는 다음을 직접 수행할 수 있다:

1. **책 기획**: 핵심 문장, 목차, 챕터 구조 제안
2. **초고 작성**: 챕터 초고, 프롤로그·에필로그, 인트로 훅
3. **퇴고 지원**: 문장 다듬기, 구조 재편, 가독성 개선
4. **출판사 기획안**: 투고용 기획서 완성본 작성
5. **제목 브레인스토밍**: 20~30개 후보 제목 생성
6. **마케팅 문구**: 책 소개문, SNS 카드뉴스 문구, 보도자료
7. **챕터별 성경 연구**: 본문 분석, 관련 구절 제안
8. **삽화 기획**: 챕터별 이미지 슬롯(훅용 일러·사진 설명), 캡션 초안, AI 프롬프트 초안(저자가 외부 툴에 넣기 전 단계)
